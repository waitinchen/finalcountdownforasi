# 🔍 服務器啟動檢查指南

## 當前狀態

服務器應該正在重新啟動中。請按照以下步驟檢查：

## ✅ 檢查步驟

### 1. 檢查服務器是否運行

在 PowerShell 中執行：
```powershell
netstat -ano | findstr :3000
```

如果看到 `LISTENING`，表示服務器正在運行。

### 2. 檢查 Node 進程

```powershell
Get-Process -Name node
```

應該會看到至少一個 Node 進程。

### 3. 測試連接

在瀏覽器中訪問：
- http://localhost:3000
- http://127.0.0.1:3000

### 4. 如果仍然無法連接

#### 方法 A: 手動啟動（推薦）

在新的 PowerShell 終端中執行：

```powershell
cd C:\Users\waiti\finalcountdownforasi\asi-countdown-dashboard

# 停止所有 Node 進程
Get-Process -Name node -ErrorAction SilentlyContinue | Stop-Process -Force

# 清理緩存
Remove-Item -Recurse -Force .next -ErrorAction SilentlyContinue

# 啟動開發服務器
npm run dev
```

**重要**：保持終端窗口打開，查看輸出信息。

#### 方法 B: 檢查端口占用

```powershell
# 查看 3000 端口占用情況
netstat -ano | findstr :3000

# 如果被其他程序占用，可以：
# 1. 停止占用端口的程序
# 2. 或使用其他端口（Next.js 會自動選擇 3001, 3002...）
```

#### 方法 C: 檢查防火牆

Windows 防火牆可能阻擋了連接。可以：
1. 暫時關閉防火牆測試
2. 或添加 Node.js 到防火牆例外

## 🐛 常見問題

### 問題 1: 端口被占用

**症狀**：服務器啟動失敗，提示端口被占用

**解決**：
```powershell
# 查找占用 3000 端口的進程
netstat -ano | findstr :3000

# 停止該進程（替換 PID 為實際進程 ID）
Stop-Process -Id <PID> -Force
```

### 問題 2: 構建錯誤

**症狀**：服務器啟動時出現編譯錯誤

**解決**：
```powershell
# 清理並重新構建
Remove-Item -Recurse -Force .next, node_modules -ErrorAction SilentlyContinue
npm install
npm run build
npm run dev
```

### 問題 3: 緩存問題

**症狀**：修改代碼後不生效

**解決**：
```powershell
# 清理 Next.js 緩存
Remove-Item -Recurse -Force .next -ErrorAction SilentlyContinue
npm run dev
```

## 📋 成功啟動的標誌

當服務器成功啟動時，終端會顯示：

```
▲ Next.js 15.x.x
- Local:        http://localhost:3000
- Ready in X.XXs
```
